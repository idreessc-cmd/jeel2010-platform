# منصة جيل 2010 التعليمية - MVP Frontend Prototype (React + Vite)

منصة تعليمية عربية متكاملة مخصصة لطلاب الصف الأول الثانوي (جيل 2010)، تم بناؤها باستخدام **React** و **Vite** لتوفير تجربة سريعة وتفاعلية بالكامل، مع محاكاة كاملة للخدمات البرمجية لتسهيل ربطها بقاعدة بيانات **Firebase** لاحقاً.

---

## 🚀 التقنيات المستخدمة (Tech Stack):
* **React 18** (مكتبة واجهة المستخدم)
* **Vite** (أداة التطوير والبناء الفائقة السرعة)
* **React Router DOM 6** (التنقل بين الصفحات)
* **Vanilla CSS (Variables)** (نظام تصميم مرن وجميل بالكامل)
* **RTL & Cairo Font** (دعم كامل ومباشر للغة العربية)
* **LocalStorage** (لحفظ درجات الامتحانات والدروس المكتملة وحالة الطلاب)

## 🎥 استراتيجية حماية الفيديوهات (YouTube Unlisted Strategy):
* **مرحلة الـ MVP:** نستخدم مشغل يوتيوب غير المدرج (YouTube Unlisted) كخيار بث تجريبي ومجاني مؤقت.
* **إخفاء الهوية (تعمية بصرية فقط):** تم حجب شعار يوتيوب، الرابط المباشر، زر المشاركة، وأكواد النسخ من خلال تضمين وسوم Modest Branding، بالإضافة لتغطية الأطراف العلوية والسفلية بـ (Transparent HTML Overlays) تمنع الطالب من النقر لفتح الفيديو في موقع يوتيوب.
* **ملاحظة إنتاجية هامة:** إن التغطية البصرية الحالية هي **تعمية في واجهة المستخدم فقط (UI Obscuring) وليست حماية تقنية أو تشفيرية حقيقية**، حيث يمكن لأي طالب خبير بـ DevTools استخراج رابط الفيديو من الشبكة. عند الإطلاق الرسمي للإنتاج، نوصي باستبدال المعرف بـ **Bunny Stream** أو **Vimeo Private** أو **Cloudflare Stream** التي تقدم تشفيراً حقيقياً وحماية متطورة ورموز وصول مشفرة (Signed Tokens).

## ⚙️ كيفية تشغيل المشروع محلياً (How to Run):
يتطلب تشغيل المشروع وجود بيئة **Node.js** مثبتة على جهازك:

1. **تثبيت التبعيات (إذا لم تكن مثبتة):**
   ```bash
   npm install
   ```
2. **تشغيل خادم التطوير المحلي:**
   ```bash
   npm run dev
   ```
3. **فتح التطبيق:**
   افتح الرابط المحلي الموضح في الطرفية (عادة ما يكون `http://localhost:5173`).

## 🛠️ كيفية التحكم بحالة الاشتراك وتجربته:
يمكنك محاكاة وتغيير حالة اشتراك أي طالب بسهولة عبر طريقتين:
1. **لوحة الإدارة (Admin Dashboard):** اضغط على "لوحة الإدارة" في القائمة العلوية للهيدر، وستجد جدولاً بالطلاب المسجلين مع زر تفعيل/إلغاء تفعيل بنقرة واحدة.
2. **عن طريق LocalStorage:** قم بفتح أدوات المطور (Developer Tools) بالمتصفح، ثم قسم Application ثم Local Storage، وقم بتعديل حقل `subscriptionStatus` لأي بريد مسجل داخل المفتاح `jeel2010_students`.

## 🔐 كيفية ربط Firebase مستقبلاً (Firebase Migration Guide):
تم تصميم بنية الخدمات البرمجية (`src/services/`) بشكل معزول تماماً عن مكونات الواجهة الرسومية (UI Components) بحيث تستقبل وتخزن البيانات عبر واجهات برمجية واضحة:
* **خدمة المصادقة authService:** تعتمد على LocalStorage حالياً، ولربطها بـ Firebase Auth ستحتاج فقط لاستبدال الدوال بـ `signInWithEmailAndPassword` و `signOut` من Firebase SDK.
* **خدمة البيانات والتقدم progressService / quizService:** لربطها بـ Firestore، يتم تبديل تخزين الـ LocalStorage بطلبات كتابة وقراءة من مستندات المستخدمين في قواعد البيانات (مثال: `doc(db, "users", email)`).
